# #ai-instruction-review Instruction review

- The review engine's second application (#ai-review-engine): run the same roles over an **instruction set** itself (here, `instructions/` plus the generated output of `node bin/cli.js --stdout`), each role proposing missing or weak rules through its lens, instead of raising code issues.
- A round is always a **full audit** (no diff variant); it **opens by running the ownership coverage lint**, turning any orphan or double-owned `#tag` into the round's first proposal, since an unowned rule is one no lens would cover.
- Each role emits `new-rule` / `strengthen` / `rehome` / `reowner` proposals through the **per-lens rubric** (clarity, terseness, efficiency, enforceability, coverage in its domain); a per-proposal verify confirms the gap is real and not already covered by a live `#tag`; an editor reduce deduplicates and reconciles ownership to keep every tag single-owned.
- The round **closes on a dimension scorecard**: the reduce consolidates the per-lens verdicts and runs the once-only **structural rubric** (cohesiveness, self-reference, lean-split, normative voice) plus a mechanical-nits sweep, then **presents a verdict per rubric dimension** -- so no dimension is silently skipped. The rubric is the one shared with the single-agent fallback `prompts/review-instructions.md` (#swe-reuse).
- It **proposes, then triages via an editable worksheet** -- the reduce writes a per-machine triage worksheet and a separate `/instruction-apply` step applies the human's recorded decisions; it never edits instruction sources except through that human-gated apply. The open queue (proposals and drafts), scorecard, and nits are ephemeral (presented, written under `.agentsmith/`, never committed); unadopted rule text is not published. The single committed output is the decisions log `docs/instruction-rules-decisions.md` -- one line per rejected/folded/deferred judgment (no draft blocks), so future rounds neither re-litigate nor prematurely re-raise. Each verified proposal is dispositioned in the worksheet -- `park` (re-surfaces next round), `reject`/`fold`/`defer` (to the decisions log), or `adopt` (into `instructions/`, via the normal change discipline #swe-done, never a blind paste); adoption stays gated on an explicit human accept.
- Role **participation** is per-application: a lens active for code review may be inactive here (e.g. `correctness` audits code, not rules).
