# #ai-instruction-review Instruction review

- The review engine's second application (#ai-review-engine): run the shared round (canonically `docs/reference-spec/review-board-protocol.md`) over an **instruction set** itself (here, `instructions/` plus the generated output of `node bin/cli.js --stdout`), each role proposing missing or weak rules through its lens, instead of raising code issues; the maintainer is `ai-engineer`.
- A round is always a **full audit** (no diff variant); it **opens by running the ownership coverage lint**, turning any orphan or double-owned `#tag` into the round's first proposal, since an unowned rule is one no lens would cover.
- Board-specific lens output: each role emits `new-rule` / `strengthen` / `rehome` / `reowner` proposals through the **per-lens rubric** (clarity, terseness, efficiency, enforceability, coverage in its domain); the shared round's verify confirms the gap is real and not already covered by a live `#tag`, and the `ai-engineer` reduce deduplicates and reconciles ownership to keep every tag single-owned.
- The round **closes on a dimension scorecard**: the reduce consolidates the per-lens verdicts and runs the once-only **structural rubric** (cohesiveness, self-reference, lean-split, normative voice) plus a mechanical-nits sweep, then **presents a verdict per rubric dimension**.
  The rubric is shared with the single-agent fallback at `prompts/review-instructions.md` (part of the authoring bundle under `prompts/`, not under `instructions/`; #swe-reuse).
- It **proposes, then triages via an editable worksheet** at `.agentsmith/instruction-review/triage.json` (gitignored, per-machine, never committed) -- the reduce writes or refreshes this file and a separate `/instruction-apply` step applies the human's recorded decisions; it never edits instruction sources except through that human-gated apply. 
  The open queue, scorecard, and nits are ephemeral. The single committed output is the decisions log `docs/instruction-rules-decisions.md` -- one line per rejected/folded/deferred judgment, so future rounds neither re-litigate nor prematurely re-raise. Each verified proposal is dispositioned -- `park` (re-surfaces), `reject`/`fold`/`defer` (to the log), or `adopt` (into `instructions/` via #swe-done).
- Role **participation** is per-application: a lens active for code review may be inactive here (e.g. `correctness` audits code, not rules).
