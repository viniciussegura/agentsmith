# #ai-status Work-status visibility

In any multi-step run -- a multi-wave plan, a batch of tasks, or a subagent fan-out -- keep the current position visible: at each wave or task transition, state which one is active and how many remain.

- Surface it on the todo mechanism where one exists; the status itself is signal, not narration (#ai-conversational).
- Single-step work needs no such ceremony.
