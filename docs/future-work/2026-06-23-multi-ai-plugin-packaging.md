# Multi-AI plugin packaging + marketplace auto-publish

Date: 2026-06-23

## What

The Claude tool set is now packaged as a Claude Code plugin (`tools/claude/.claude-plugin/plugin.json` + a root `marketplace.json`, generated by `bin/build-plugin.js`). Two follow-ons are deferred:

1. **Other AIs.** Only `tools/claude/` exists. A future `tools/gemini/` (etc.) would need its own packaging — Gemini/Codex do not consume Claude Code plugins, so each tool's namespacing/install story is per-tool.
2. **Marketplace auto-publish.** The plugin version is bumped by hand in `package.json` and the manifests regenerated via `npm run build:plugin`; there is no CI release that tags + regenerates + verifies on publish.

## Why it matters

The plugin is the Claude-only distribution; the `tools/<ai>/` mechanism remains the AI-neutral source of truth. Until a second AI target is concrete, multi-AI packaging would be speculative (see `#swe-future-work`). Auto-publish is ergonomics, not correctness.

## Constraints / dependencies

- Plugins are Claude Code-specific; the abstraction (skills/commands/agents/hooks) is largely Claude-shaped. ChatGPT especially has no filesystem-skill model — only the instruction text (`AGENTS.md`) ports there.
- Defer until there is a concrete second-tool target; do not speculatively encode unverified plugin formats.
- Auto-publish would need a release workflow that runs `build:plugin`, fails on drift (the existing guard test), and tags — pairs naturally with the existing version-in-`package.json` convention.
